# BillDesk Java POS Billing System

BillDesk is a small full-stack point-of-sale billing application built with plain Java Servlets, JDBC, MySQL, vanilla HTML/CSS/JavaScript, Maven, and a Tomcat WAR deployment. It is intentionally framework-light so the request flow is visible from browser to servlet to DAO to database.

## Current Runtime Target

| Tool | Version | Notes |
|------|---------|-------|
| Java | 21 | Maven compiles with `--release 21`. |
| Maven | 3.6+ | Builds the deployable WAR. |
| Tomcat | 11.x | Provides Jakarta Servlet 6.1 at runtime. |
| MySQL | 8.x | Stores products and checkout transactions. |

Tomcat 11 implements Jakarta Servlet 6.1, so the project uses `jakarta.servlet.*` imports and declares `jakarta.servlet-api` as a `provided` dependency. Do not deploy this WAR to Tomcat 9 or older; those servers use the legacy `javax.servlet.*` namespace.

## What Changed In The Cleanup

- Updated Servlet API alignment for Tomcat 11: `jakarta.servlet-api` is now `6.1.0`, and `web.xml` uses the Servlet 6.1 descriptor.
- Removed duplicate receipt overlay markup and duplicate JavaScript receipt handlers.
- Changed frontend API URLs to context-relative values (`get-product`, `checkout`) so the app works when deployed as `/billing-system`.
- Set `MOCK_MODE` to `false` for normal Tomcat/MySQL operation.
- Replaced deprecated DBCP timeout setters with `Duration` based setters.
- Added a null/blank guard to `ProductDAO.exists`.
- Moved module READMEs into `docs/`.
- Added `.gitignore` entries for generated Maven artifacts.

## Setup

1. Create the database.

```bash
mysql -u root -p < src/main/resources/schema.sql
```

2. Configure the database connection.

Edit `src/main/resources/db.properties` for your local MySQL credentials:

```properties
db.url=jdbc:mysql://localhost:3306/billdesk?useSSL=false&serverTimezone=Asia/Kolkata&allowPublicKeyRetrieval=true
db.username=root
db.password=your_password_here
```

This project includes `db.properties` for local development. Do not commit real production credentials to a shared repository.

3. Build the WAR.

```bash
mvn clean package
```

The WAR is created at:

```text
target/billing-system.war
```

4. Deploy to Tomcat 11.

Copy the WAR to Tomcat's `webapps` directory:

```bash
cp target/billing-system.war $CATALINA_HOME/webapps/
```

Then open:

```text
http://localhost:8080/billing-system/
```

## Frontend Mock Mode

Normal deployment uses the Java backend:

```javascript
MOCK_MODE: false
```

For standalone frontend testing without Tomcat or MySQL, temporarily set `MOCK_MODE: true` in `src/main/webapp/script.js`, open `src/main/webapp/index.html`, and scan one of:

```text
P001 P002 P003 P004 P005 P006
```

Switch it back to `false` before building the WAR for Tomcat deployment.

## Main Endpoints

| Endpoint | Servlet | Purpose |
|----------|---------|---------|
| `GET /get-product?uid=P001` | `ProductServlet` | Looks up an active product by UID and returns JSON. |
| `POST /checkout` | `CheckoutServlet` | Saves the sale and streams a PDF receipt. |

The frontend uses context-relative URLs (`get-product`, `checkout`). When deployed at `/billing-system`, those resolve correctly under the application context.
